───────────────────────────────────────────────────────────────────
  IF SOMETHING GOES WRONG
───────────────────────────────────────────────────────────────────

PROBLEM: "Repository name already exists"
  → Use a different name like quatic-simulator or quatic-thesis

PROBLEM: Upload to GitHub takes forever / freezes
  → Try uploading in two batches: first the .py files, then the folders

PROBLEM: Streamlit says "App failed to start"
  → Click "Manage app" (bottom-right of the deploy page)
  → Look at the logs to see the error
  → Common cause: missing requirements.txt — make sure it was uploaded
  → Copy the error and send it to Claude, we'll fix it

PROBLEM: "Authorize Streamlit" doesn't work
  → Log out of GitHub, log back in, try again
  → Make sure pop-ups aren't blocked in your browser

PROBLEM: The app is slow to load the first time
  → Normal. Streamlit's free tier "sleeps" inactive apps. First load
    after a long pause takes ~30 seconds. Subsequent loads are instant.
